# Options Flow Analytics Platform

A full-stack analytics platform for high-frequency options Market-By-Order (MBO) data. Ingests ES futures and SPX cash options data, stores it in Parquet format, and renders 12 professional visualizations for quantitative analysis, flow detection, and greeks monitoring.

## Features

| Page | Visualizations |
|------|---------------|
| **Dashboard** | KPI row, ES vs SPX basis chart, pulling/stacking event ticker |
| **Order Flow** | Volume heatmap, MBO size histogram (log scale), bid-ask spread, order book depth waterfall |
| **Signals** | Pulling/stacking events on price, delta-adjusted net flow (stacked area), ES-SPX lead/lag analysis |
| **Greeks** | 3D surface (strike × DTE × greek), vanna/charm exposure bars, theta decay family, vomma vs vega scatter |
| **Analytics** | Cross-greek Pearson correlation matrix with click-to-drilldown, custom query builder + export |

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Data pipeline | Polars + PyArrow |
| Query engine | DuckDB (SQL on Parquet) |
| Backend API | FastAPI (Python) |
| Frontend | React 18 + TypeScript + Vite |
| Styling | Tailwind CSS |
| Charts | Recharts + Plotly.js |
| State | Zustand + TanStack Query |
| Deployment | Docker + Nginx / GitHub Pages |

## Quick Start

### Prerequisites
- Python 3.10+
- Node.js 18+

### 1. Install dependencies

```bash
# Python (pipeline + API)
pip install polars pyarrow fastapi "uvicorn[standard]" duckdb

# Frontend
cd frontend && npm install
```

### 2. Ingest data

Drop CSV files into `data/raw/` then run:

```bash
cd pipeline
python ingest.py --input "../data/raw/*.csv" --output "../data/flow.parquet"
```

The script deduplicates on `(timestamp, Side, future_strike)` and appends safely — already-processed files are tracked in `processed_files.log`.

```bash
# Run schema unit tests
python -m pytest tests/ -v
```

### 3. Start the API

```bash
# From project root
PARQUET_PATH=data/flow.parquet python -m uvicorn api.main:app --host 0.0.0.0 --port 8000 --reload
```

API docs available at `http://localhost:8000/docs`

### 4. Start the frontend

```bash
cd frontend
npm run dev      # dev server on http://localhost:3000
npm run build    # production build → dist/
```

The Vite dev server proxies `/api/*` → `http://localhost:8000` automatically.

## Docker Deployment

Runs the full stack (API + frontend) in a single compose:

```bash

docker compose up --build
```

- Frontend: `http://localhost:80`
- API: `http://localhost:8000`

## GitHub Pages Deployment

The frontend can be deployed to GitHub Pages as a static site. The API must be hosted separately (e.g., a VPS, Railway, Render, or run locally).

### Setup

1. Fork or push this repo to GitHub.

2. Set the repository secret `VITE_API_URL` to your hosted API URL (e.g. `https://your-api.example.com`):
   - Go to **Settings → Secrets and variables → Actions → New repository secret**
   - Name: `VITE_API_URL`
   - Value: your FastAPI base URL (no trailing slash)

3. Enable GitHub Pages:
   - Go to **Settings → Pages**
   - Source: **GitHub Actions**

4. Push to `main` — the workflow in `.github/workflows/deploy.yml` will build and deploy automatically.

The live site will be available at `https://<username>.github.io/<repo-name>/`

## API Reference

All endpoints accept optional query parameters for filtering:

| Parameter | Type | Description |
|-----------|------|-------------|
| `start` | datetime string | Filter rows after this timestamp |
| `end` | datetime string | Filter rows before this timestamp |
| `side` | `Bid` or `Ask` | Filter by order book side |
| `strike_min` / `strike_max` | float | SPX strike range |
| `dte_min` / `dte_max` | float | Time to expiry range (in years) |
| `bucket_minutes` | int | Time bucket size (1, 5, 15, 30, 60) |

### Endpoints

```
GET /signals/kpi               Dashboard KPI summary
GET /signals/basis             ES vs SPX basis per time bucket
GET /signals/pulling_stacking  P/S events overlaid on price
GET /signals/net_flow          Delta-adjusted net flow per bucket

GET /volume/heatmap            Time bucket × strike × MBO sum
GET /volume/histogram          Exploded MBO order sizes
GET /volume/spread             Bid-ask spread vs SPX price
GET /volume/depth              Order book depth by strike and side

GET /greeks/surface            3D surface (strike × DTE × greek)
GET /greeks/vanna_charm        Vanna or charm exposure by strike
GET /greeks/theta              Theta decay curves by strike
GET /greeks/scatter            Vomma vs Vega scatter
GET /greeks/correlation        Pearson correlation matrix (all 16 greeks)
```

## Domain Glossary

| Term | Description |
|------|-------------|
| **MBO** | Market-By-Order — individual order sizes at each price level, more granular than MBP |
| **Pulling** | Rapid order cancellation; clusters often precede a price drop |
| **Stacking** | Rapid order addition; clusters often precede a rally |
| **ES-SPX Basis** | ES futures typically lead SPX cash by 2–5 seconds; divergence signals an imminent move |
| **Gamma exposure** | Short-gamma dealers hedge dynamically, amplifying directional moves |
| **Vanna** | dDelta/dVol — IV moves cause automatic delta hedging flows |
| **Charm** | dDelta/dTime — important near expiry as time decay shifts the hedge ratio |
| **Vomma** | dVega/dVol — high vomma = nonlinear sensitivity to volatility regimes |
| **Theta** | Daily dollar cost of holding a long option; accelerates after 21 DTE |
